Hearth release "{{ .Release.Name }}" ({{ .Chart.AppVersion }}) is installed in namespace "{{ .Release.Namespace }}".

Verify the operator deployment:
  kubectl get deploy -n {{ .Release.Namespace }} {{ include "hearth.fullname" . }}-controller-manager

The operator deploys a data-plane gateway per LLMService using:
  {{ include "hearth.gatewayImage" . }}

Prerequisite for scale-to-zero (Hearth degrades gracefully without it):
  - KEDA (required for scale-to-zero / autoscaling):
      https://keda.sh/docs/2.20/deploy/

Prometheus and Grafana are independent, opt-in integrations. Hearth does not
create their resources; see examples/observability in the project repository.

Try the physically validated NVIDIA A10 profile (requires an NVIDIA device
plugin, nvidia.com/gpu.product=NVIDIA-A10, and a default StorageClass with
at least 30 GiB available):
  1. Create a workload namespace:
       kubectl create namespace ai --dry-run=client -o yaml | kubectl apply -f -
  2. Deploy the device profile:
       kubectl apply -n ai -k "https://github.com/hearth-project/hearth//examples/nvidia/a10?ref=v{{ .Chart.AppVersion }}"
  3. Watch it:
       kubectl get llmservice -n ai -w

Note: serving real tokens requires a node with a supported accelerator
(e.g. an NVIDIA GPU + device plugin). Without one, the backend stays Pending.
